## 4) Top 5 recommendations for next execution wave

1. **Close stability blockers first (TPA-026 scope) before any feature expansion**  
   Resolve compile/runtime P0s (duplicate class conflict, missing usings, enemy damage flow, null guards) to restore a reliable baseline.

2. **Execute multiplayer spine immediately as Wave priority #1**  
   Implement `PM2-100` create/join/return loop, then lock host-authoritative boundaries for projectile/puzzle/pickup resolution (`PM2-102/103/104/105`).

3. **Ship diagnostics before broader integration**  
   Deliver TickId/EventId + host/client state-hash probes (`PM2-001`) and run lag/packet-loss soak harness (`PM2-106`) to prevent hidden desync debt.

4. **Stand up platform CI/profile matrix now, not later**  
   Implement WebGL/mobile/Xbox smoke matrix + quality/tick profile scaffolding (`PM2-002`, `PM2-202`) to catch cross-platform regressions early.

5. **Use explicit gate criteria to control scope creep**  
   Run a formal M1-style gate check (`PM2-107` logic): no new content wave until 2P co-op survives stability criteria with diagnostics evidence.
